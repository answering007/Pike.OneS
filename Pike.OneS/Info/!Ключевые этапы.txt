Реализоваынные провайдеры:
1) Ado.Net provider на основе "обертки" над COM-компонентом 1С (Late binding)
2) Web-сервис 1C (Power Query)
3) Ado.Net provider на основе Web-сервис 1C (для увеличения скорости работы объемных запросов)
Дополнительно реализован пример использования Odata-интерфейса в Power Query

Ключевые этапы интеграции и использования Ado.Net провейдера 1С и Web-сервиса:

1) Создание web-сервиса WebIntegration с методом GetQueryResult, возвращающий текстовый аргумент и принимающим текстовый параметр QueryText:
Функция GetQueryResult(QueryText)
	//Имя вэб-сервиса: WebIntegration
	Запрос = Новый Запрос(QueryText);
	ТаблицаРезультатов  = Запрос.Выполнить().Выгрузить();
	
	ЗаписьXML = Новый ЗаписьXML;
	ЗаписьXML.УстановитьСтроку();
	// Запись документа XML
	СериализаторXDTO.ЗаписатьXML(ЗаписьXML, ТаблицаРезультатов);
	СтрокаXml = ЗаписьXML.Закрыть();
	Возврат СтрокаXml;
КонецФункции

2) Создание роли для использования средств интеграции. Роль "Интегратор":
а) чтение объектов (которые планируются к использованию в запросах - поставить флаг):
	-константы
	-справочники
	-документы
	-журналы документов
	-планы видов характеристик
	-планы счетов
	-планы видов расчета
	-регистры сведений
	-регистры накопления
	-регистры бухгалтерии
	-регистры расчета
б) Общие->Web-сервисы->WebIntegration->GetQueryResult - использование (поставить флаг)
в) Общие->Параметры сеанса->ТекущийПользователь - Получение (поставить флаг)
г) Общие->Параметры сеанса->ТекущийВнешнийПользователь - Получение (поставить флаг)
д) Права: Внешнее соединение

3) Создание пользователя Integration:
	-Аутоидентификация 1С (Имя (предпочтительно латинницей): Integration, Пароль: [ПарольК1С])
	-Роль: Интегратор

Для проверки провайдера Ado.Net на основе "обертки" над COM-компонентом 1С можно использовать Pike.OneS.Test x86.exe, указав в качестве параметра строку подключения к 1С. Пример строки подключения с указанием ProgId доступен в приложении Pike.OneS.Test x86.exe

Для использования Ado.Net провайдера из приложения x64, COM-компонент 1С необходимо "обернуть" в службу COM+. Инструкция для создания "обертки" доступна в файле "Вызов 32-битных COM-объектов на стороне 64-битного сервера.html"
Примерный результат в виде скрина доступен в файле "COM+.png"
